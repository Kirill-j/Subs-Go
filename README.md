# SubsControl

**SubsControl** — fullstack-проект для учета и управления платными подписками. 
Пользователи могут регистрироваться в системе, добавлять свои подписки, редактировать их и визуализировать статистику расходов.

## Быстрый старт

Проект можно развернуть локально, используя редактор VS Code. 
Серверная часть и клиентская запускаются в раздельных окнах терминала. 
Подробные инструкции описаны в разделах запуска ниже.

### 1) Клонировать репозиторий
```bash
git clone <your-repo-url>
cd subs_control
```

### 2) Поднять MySQL
Создай пустую базу:
```bash
CREATE DATABASE subscription_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 3) Запуск Backend
Для запуска бэкенда откройте терминал в VS Code (в корневой папке или перейдя в папку бэкенда) и выполните:
```bash
go run main.go
```

### 4) Запуск Frontend
В том же VS Code откройте новую вкладку терминала, перейдите в папку frontend и выполните две команды:
```bash
npm install
npm start
```

## База данных, миграции, сиды

В проекте используется реляционная база данных MySQL.
Создание таблиц и их обновление (миграции) происходит автоматически при
запуске приложения с помощью встроенных механизмов ORM-библиотеки GORM. 
Никаких дополнительных SQL-скриптов накатывать не нужно.

### Запуск Backend
Из `apps/backend`:

```bash
go run .
```

Backend поднимается на:

 - http://localhost:8080

### Запуск Frontend
Из `apps/frontend`:

```bash
npm install
npm run dev
```

## API (эндпоинты)

Взаимодействие фронтенда с бэкендом происходит через следующие REST API эндпоинты (с поддержкой CORS):

### Public
 - `POST /api/public/register` — Регистрация нового пользователя

 - `POST /api/public/login` — Авторизация пользователя и получение JWT токена

### Private

- `GET /api/users/me` — Получение данных текущего авторизованного пользователя

- `PUT /api/users/update` — Обновление данных профиля

- `GET /api/subscription` — Получение списка подписок пользователя

- `POST /api/subscription/add` — Добавление новой подписки

- `PUT /api/subscription/edit/:id` — Редактирование существующей подписки по её ID

- `DELETE /api/subscription/delete/:id` — Удаление подписки по её ID

## Частые ошибки и решения
### 1) “Статус пустой / не обновляется”
Обычно это:
 - старая БД без дефолтов
 - миграции применились не так, как ожидаешь

Решение:
 - удалить базу `bookinghub`
 - создать заново
 - запустить backend (миграции заново)

### 2) 404 на эндпоинты
Проверить:
 - backend запущен на правильном порту
 - фронт ходит туда же (прокси/база URL)
 - маршрут реально существует в `main.go`

### 3) После логина UI не обновляется до refresh

Почти всегда проблема в том, что `me` или `token` не обновляются сразу в state.
Решение: после логина сохранять token, обновлять `me` и триггерить загрузку публичных данных (`loadPublic`).

### 4) “Cannot add or update a child row: a foreign key constraint fails”

Это про внешние ключи в MySQL.
Чаще всего сиды вставляются в неверном порядке или owner_user_id ссылается на несуществующего пользователя.
Лечится исправлением сидов или пересозданием БД.
