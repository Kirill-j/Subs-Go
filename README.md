# SubsControl

**SubsControl** — fullstack-проект для учета и управления платными подписками. 
Пользователи могут регистрироваться в системе, добавлять свои подписки, редактировать их и визуализировать статистику расходов.

## Быстрый старт

Проект можно развернуть локально, используя редактор VS Code. 
Серверная часть и клиентская запускаются в раздельных окнах терминала. 
Подробные инструкции описаны в разделах запуска ниже.

### 1) Клонировать репозиторий
```bash
git clone <your-repo-url>
cd subs_control
```

### 2) Поднять MySQL
Создай пустую базу:
```bash
CREATE DATABASE subscription_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 3) Запуск Backend
Для запуска бэкенда откройте терминал в VS Code (в корневой папке или перейдя в папку бэкенда) и выполните:
```bash
go run main.go
```

### 4) Запуск Frontend
В том же VS Code откройте новую вкладку терминала, перейдите в папку frontend и выполните две команды:
```bash
npm install
npm start
```

## Переменные окружения
Backend читает `.env` (через `godotenv`), либо переменные окружения системы.

Пример `.env` для `subscription-backend-go/.env`:

```env
DB_USER=root
DB_PASSWORD=
DB_NAME=subscription_db
DB_HOST=127.0.0.1
DB_PORT=3306
PORT=8080
JWT_SECRET=my_super_secret_key_123
```

## База данных, миграции, сиды

В проекте используется реляционная база данных MySQL.
Создание таблиц и их обновление (миграции) происходит автоматически при
запуске приложения с помощью встроенных механизмов ORM-библиотеки GORM. 
Никаких дополнительных SQL-скриптов накатывать не нужно.

## API (эндпоинты)

Взаимодействие фронтенда с бэкендом происходит через следующие REST API эндпоинты (с поддержкой CORS):

### Public
 - `POST /api/public/register` — Регистрация нового пользователя

 - `POST /api/public/login` — Авторизация пользователя и получение JWT токена

### Private

- `GET /api/users/me` — Получение данных текущего авторизованного пользователя

- `PUT /api/users/update` — Обновление данных профиля

- `GET /api/subscription` — Получение списка подписок пользователя

- `POST /api/subscription/add` — Добавление новой подписки

- `PUT /api/subscription/edit/:id` — Редактирование существующей подписки по её ID

- `DELETE /api/subscription/delete/:id` — Удаление подписки по её ID

## Частые ошибки и решения
### 1) Бэкенд не запускается (ошибка подключения к БД)
При выполнении `go run main.go` в терминале появляются ошибки вида `failed to connect to database или dial tcp: connect: connection refused.`
Причина: Сервер MySQL не запущен, либо указаны неверные доступы.

Решение:
 - Убедитесь, что служба MySQL запущена на вашем компьютере.
 - Проверьте файл .env (или конфигурацию в коде): правильность логина, пароля, порта (обычно 3306) и имени базы данных.
 - Убедитесь, что база данных, указанная в конфигурации, предварительно создана в MySQL (GORM создает таблицы, но саму базу данных нужно создать вручную, например командой `CREATE DATABASE subscriptions_db;`).

### 2) Ошибка "Port is already in use"
При запуске бэкенда или фронтенда терминал выдает ошибку, что порт (например, 8080 или 3000) уже занят bind: address already in use.
Причина: Предыдущая сессия сервера не была завершена корректно, или на этом порту работает другое приложение.

Решение: 
- В VS Code нажмите иконку корзины в терминале, чтобы убить зависший процесс, и откройте новый терминал.
- Если это не помогло, найдите процесс, занимающий порт, через диспетчер задач (или команду `killport 8080` / `fuser -k 8080/tcp` в Linux/Mac) и завершите его.
